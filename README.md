# Учёт оплат, проектов и закрывающих актов

Мини-система для digital-агентства: связывает **проект / юрлицо → оплаты → этапы
работ → статус закрывающих актов**. Из банковской выписки строит понятную картину:
кто заплатил, по какому проекту и этапу, отправлен ли акт, подписан ли, где долги по
закрытию и что требует внимания.

> Тестовое задание. Исходное ТЗ — [`docs/brief/TASK.md`](docs/brief/TASK.md),
> мок-данные — банковская выписка в [`docs/brief/`](docs/brief/).
> Подробный план разработки и архитектура — [`docs/PLAN.md`](docs/PLAN.md).

## Стек

- **Backend:** Laravel 12 (PHP 8.4), REST API
- **Frontend:** Vue 3 SPA (Vite + Pinia + Tailwind)
- **БД:** PostgreSQL 16 (тесты — SQLite)
- **Инфраструктура:** Docker Compose

## Запуск

```bash
docker compose up -d        # поднимет БД, backend, frontend; миграции и сиды — автоматически
```

Точные команды и порты будут добавлены по мере сборки.

## Архитектура (кратко)

Данные → бизнес-логика → отображение разнесены по слоям:

```
PDF-выписка → парсер+классификатор → сырой слой (все операции)
            → доменный слой (клиенты, проекты, оплаты, акты)
            → доменные сервисы (статусы актов, итоги, фильтры)
            → REST API → Vue SPA (дашборд)
```

Ключевая идея: не каждая операция выписки — это оплата клиента за проект. Система
отделяет выручку по проектам от шума (налоги, зарплаты, аренда, комиссии, субподряд),
а затем считает итоги и статусы закрывающих актов. Подробности, сущности, связи и
принятые допущения — в [`docs/PLAN.md`](docs/PLAN.md) (переедут сюда по мере готовности).
